# 安全政策

## 回報安全漏洞

我們非常重視 21dev.org 的安全性。如果你發現任何安全漏洞，請負責任地向我們回報。

### 回報方式

**請勿**在公開的 GitHub Issues 中回報安全漏洞。

請通過以下方式回報：

1. **GitHub Security Advisories**（推薦）
   - 前往 [Security Advisories](https://github.com/21dev-org/21dev-org/security/advisories)
   - 點擊 "Report a vulnerability"

2. **私訊聯繫**
   - 透過 [Signal 群組](https://signal.group/#CjQKIKEW5NtKmgLKX9JOei4GOaTCl1enEIOpFC1R6vjm5_RCEhD3TXZew1_1HSLkD9sh8r5K) 私訊維護者

### 回報內容

請在回報中包含：

- 漏洞的詳細描述
- 重現步驟
- 潛在影響
- 可能的修復建議（如果有的話）

### 回應時間

- 我們會在 **48 小時內**確認收到你的回報
- 我們會在 **7 天內**提供初步評估
- 我們會持續更新修復進度

### 安全更新

安全更新會透過以下方式發布：

- GitHub Releases
- 網站公告（重大漏洞）

## 支援的版本

由於 21dev.org 是持續部署的網站，我們只支援最新版本。

## 感謝

我們感謝所有負責任地回報安全問題的研究人員和社群成員。
