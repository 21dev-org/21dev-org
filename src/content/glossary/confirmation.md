確認是指交易被包含在區塊中後，在其上方又堆疊了多少個區塊。確認數越多，交易被逆轉的可能性越低。

## 確認數計算

```
確認數 = 當前區塊高度 - 交易所在區塊高度 + 1

例如：
交易在區塊 800,000
當前區塊 800,005
確認數 = 800,005 - 800,000 + 1 = 6
```

## 安全性建議

| 確認數      | 安全性 | 適用場景     |
| ----------- | ------ | ------------ |
| 0（未確認） | 不安全 | 不建議       |
| 1           | 低     | 極小額       |
| 3           | 中     | 小額         |
| 6           | 高     | 大額交易標準 |
| 60+         | 極高   | 交易所存款   |

## 為什麼需要等待確認？

- 防止雙花攻擊
- 確保交易最終性
- 每個區塊約 10 分鐘，6 確認約需 1 小時
