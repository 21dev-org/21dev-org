比特幣是一種去中心化的數位貨幣，於 2009 年由化名中本聰（Satoshi Nakamoto）的人或團隊創建。它是第一個成功實現的加密貨幣，不依賴任何中央機構或中介即可進行點對點交易。比特幣同時是一種貨幣（bitcoin，小寫）和一個網路協議（Bitcoin，大寫）。

## 比特幣是什麼？

### 三層含義

```
比特幣的三個層面：

┌─────────────────────────────────────────────────────────┐
│                                                         │
│  1. 貨幣（bitcoin / BTC）                               │
│     - 數位原生的稀缺資產                                 │
│     - 可分割、可轉移、可驗證                             │
│     - 總量固定 2,100 萬枚                                │
│                                                         │
│  2. 網路（Bitcoin Network）                             │
│     - 全球分散式節點網路                                 │
│     - 點對點通訊協議                                     │
│     - 無需許可，任何人可加入                             │
│                                                         │
│  3. 協議（Bitcoin Protocol）                            │
│     - 共識規則                                          │
│     - 交易驗證邏輯                                       │
│     - 區塊結構標準                                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 可分割性

```
比特幣的單位：

1 BTC = 100,000,000 satoshi（聰）

單位換算：
┌─────────────────────────────────────────────────────────┐
│  單位             │  BTC 數量              │ 縮寫       │
├───────────────────┼────────────────────────┼────────────┤
│  1 bitcoin        │  1.00000000 BTC        │ BTC        │
│  1 centibitcoin   │  0.01000000 BTC        │ cBTC       │
│  1 millibitcoin   │  0.00100000 BTC        │ mBTC       │
│  1 microbitcoin   │  0.00000100 BTC        │ μBTC/bit   │
│  1 satoshi        │  0.00000001 BTC        │ sat        │
└─────────────────────────────────────────────────────────┘

實際應用：
- 大額交易：使用 BTC（如 1.5 BTC）
- 日常支付：使用 sats（如 50,000 sats）
- 閃電網路：通常以 sats 計價

有趣事實：
- 1 satoshi 是比特幣最小單位
- 如果 1 BTC = $100,000，1 sat = $0.001
- 閃電網路可處理毫聰（millisatoshi）
```

### 透明性與隱私

```
比特幣的透明性：

公開可見：
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  ✓ 所有交易記錄                                         │
│  ✓ 所有地址餘額                                         │
│  ✓ 交易時間戳                                           │
│  ✓ 交易金額                                             │
│  ✓ 區塊內容                                             │
│                                                         │
└─────────────────────────────────────────────────────────┘

隱私保護：
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  ✓ 地址不包含身份資訊                                   │
│  ✓ 可以生成無限多地址                                   │
│  ✓ 無需 KYC 即可使用                                    │
│  ✗ 交易可被追蹤分析                                     │
│  ✗ 與交易所互動會洩露身份                               │
│                                                         │
└─────────────────────────────────────────────────────────┘

隱私是假名的，不是匿名的：
- 地址本身不關聯身份
- 但交易模式可以被分析
- 一旦某地址關聯到身份，歷史可被追溯
```

## 運作原理

### 交易流程

```
一筆比特幣交易的生命週期：

1. 創建交易
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  Alice 的錢包                                           │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 輸入：之前收到的 UTXO（1 BTC）                   │    │
│  │ 輸出：                                          │    │
│  │   - Bob 的地址：0.5 BTC                         │    │
│  │   - 找零地址：0.499 BTC                         │    │
│  │ 手續費：0.001 BTC                               │    │
│  │ 簽名：用私鑰簽署                                │    │
│  └─────────────────────────────────────────────────┘    │
│                                                         │
└─────────────────────────────────────────────────────────┘
        ↓
2. 廣播交易
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  Alice 節點 ──→ 鄰近節點 ──→ 全網路                     │
│                                                         │
│  每個節點驗證：                                         │
│  - 簽名是否有效？                                       │
│  - UTXO 是否存在且未花費？                              │
│  - 金額是否正確？                                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
        ↓
3. 進入記憶體池
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  記憶體池（Mempool）                                    │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 等待確認的交易池                                 │    │
│  │ 交易按手續費率排序                               │    │
│  │ 礦工從這裡選擇交易                               │    │
│  └─────────────────────────────────────────────────┘    │
│                                                         │
└─────────────────────────────────────────────────────────┘
        ↓
4. 礦工打包
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  礦工選擇高手續費交易                                   │
│  組成候選區塊                                           │
│  進行工作量證明計算                                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
        ↓
5. 區塊確認
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  新區塊被廣播到網路                                     │
│  交易獲得 1 次確認                                      │
│  每增加一個區塊，確認數 +1                              │
│  通常 6 次確認視為不可逆                                │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 挖礦與共識

```
工作量證明（Proof of Work）：

礦工的工作：
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  1. 收集待確認交易                                      │
│  2. 組成區塊（包含前一區塊雜湊）                        │
│  3. 尋找有效的 nonce 值                                 │
│                                                         │
│     尋找目標：                                          │
│     SHA256(區塊頭) < 目標難度                           │
│                                                         │
│     嘗試：                                              │
│     nonce=0: 0x8a3f2b... (太大)                         │
│     nonce=1: 0x9c4e1d... (太大)                         │
│     nonce=2: 0x2b8f4c... (太大)                         │
│     ...                                                 │
│     nonce=N: 0x000000000019d6... (成功！)               │
│                                                         │
│  4. 廣播新區塊，獲得獎勵                                │
│                                                         │
└─────────────────────────────────────────────────────────┘

難度調整：
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  目標：平均每 10 分鐘產生一個區塊                       │
│                                                         │
│  每 2016 個區塊（約 2 週）調整一次                      │
│                                                         │
│  如果區塊產生太快 → 難度增加                            │
│  如果區塊產生太慢 → 難度降低                            │
│                                                         │
│  最大調整幅度：4 倍                                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 技術架構

### 區塊結構

```
比特幣區塊結構：

區塊頭（80 bytes）：
┌─────────────────────────────────────────────────────────┐
│  版本（4 bytes）                                        │
│  前一區塊雜湊（32 bytes）                               │
│  Merkle 根（32 bytes）                                  │
│  時間戳（4 bytes）                                      │
│  難度目標（4 bytes）                                    │
│  Nonce（4 bytes）                                       │
└─────────────────────────────────────────────────────────┘

區塊體：
┌─────────────────────────────────────────────────────────┐
│  交易數量                                               │
│  Coinbase 交易（礦工獎勵）                              │
│  交易 1                                                 │
│  交易 2                                                 │
│  ...                                                    │
│  交易 N                                                 │
└─────────────────────────────────────────────────────────┘

區塊限制：
- 最大區塊大小：4 MB（權重上限）
- 實際大小：通常 1.5-2 MB
- 每區塊交易數：1,500-3,000 筆
```

### 地址類型

```
比特幣地址演進：

1. Legacy (P2PKH)
┌─────────────────────────────────────────────────────────┐
│  格式：以 "1" 開頭                                      │
│  範例：1BvBMSEYstWetqTFn5Au4m4GFg7xJaNVN2                │
│  說明：最原始的地址格式                                 │
└─────────────────────────────────────────────────────────┘

2. P2SH（多簽/SegWit 兼容）
┌─────────────────────────────────────────────────────────┐
│  格式：以 "3" 開頭                                      │
│  範例：3J98t1WpEZ73CNmQviecrnyiWrnqRhWNLy                │
│  說明：支援多簽和包裝 SegWit                            │
└─────────────────────────────────────────────────────────┘

3. Native SegWit (Bech32)
┌─────────────────────────────────────────────────────────┐
│  格式：以 "bc1q" 開頭                                   │
│  範例：bc1qar0srrr7xfkvy5l643lydnw9re59gtzzwf5mdq        │
│  說明：更低手續費、更好的錯誤檢測                       │
└─────────────────────────────────────────────────────────┘

4. Taproot (Bech32m)
┌─────────────────────────────────────────────────────────┐
│  格式：以 "bc1p" 開頭                                   │
│  範例：bc1p5d7rjq7g6rdk2yhzks9smlaqtedr4dekq08ge8ztwac72sfr9rusxg3297│
│  說明：最新格式，支援 Schnorr 簽名和複雜腳本            │
└─────────────────────────────────────────────────────────┘

建議：新錢包應使用 bc1q 或 bc1p 地址
```

## 安全性

### 安全模型

```
比特幣的安全保障：

1. 密碼學安全
┌─────────────────────────────────────────────────────────┐
│  私鑰：256 位隨機數                                     │
│  可能的組合：2²⁵⁶ ≈ 10⁷⁷                               │
│  宇宙原子數：約 10⁸⁰                                    │
│  暴力破解：計算上不可能                                 │
└─────────────────────────────────────────────────────────┘

```
關於比特幣的常見誤解：

誤解 1：「比特幣是匿名的」
┌─────────────────────────────────────────────────────────┐
│  事實：比特幣是假名的（pseudonymous）                   │
│  - 所有交易公開可查                                     │
│  - 地址可被追蹤分析                                     │
│  - 與交易所互動會關聯身份                               │
└─────────────────────────────────────────────────────────┘

誤解 2：「比特幣只用於犯罪」
┌─────────────────────────────────────────────────────────┐
│  事實：                                                 │
│  - 非法活動佔比特幣交易 <1%                             │
│  - 法幣在非法活動中使用更普遍                           │
│  - 區塊鏈透明性實際上有助於追蹤犯罪                     │
└─────────────────────────────────────────────────────────┘

誤解 3：「比特幣浪費能源」
┌─────────────────────────────────────────────────────────┐
│  事實：                                                 │
│  - 約 50%+ 使用可再生能源                               │
│  - 能源成本是安全性的來源                               │
│  - 激勵使用閒置/浪費的能源                              │
│  - 與傳統金融系統比較應考慮全面成本                     │
└─────────────────────────────────────────────────────────┘

誤解 4：「比特幣沒有內在價值」
┌─────────────────────────────────────────────────────────┐
│  事實：                                                 │

│  - 無需許可的全球支付網路                               │
│  - 網路效應和林迪效應                                   │
└─────────────────────────────────────────────────────────┘

誤解 5：「政府可以關閉比特幣」
┌─────────────────────────────────────────────────────────┐
│  事實：                                                 │
│  - 去中心化設計使關閉極其困難                           │
