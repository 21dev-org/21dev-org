**計費方式**

比特幣手續費按**交易大小**計算，而非交易金額：

```
手續費 = 交易大小 (vBytes) × 費率 (sats/vB)
```

**影響交易大小的因素**

| 因素     | 影響                |
| -------- | ------------------- |
| 輸入數量 | 越多輸入，交易越大  |
| 輸出數量 | 越多輸出，交易越大  |
| 地址類型 | SegWit 比傳統地址小 |
| 簽名類型 | 多簽比單簽大        |

**典型交易大小**

| 類型                    | 大小    |
| ----------------------- | ------- |
| 1 輸入 1 輸出（P2WPKH） | ~110 vB |
| 1 輸入 2 輸出（P2WPKH） | ~140 vB |
| 2 輸入 2 輸出（P2WPKH） | ~210 vB |

**省錢技巧**

1. 使用 SegWit 地址（bc1q... 或 bc1p...）
2. 避免碎片化 UTXO
3. 低峰時段發送（週末通常較便宜）
4. 合併小額 UTXO 在低費時期

**為什麼轉 1 BTC 和轉 0.001 BTC 費用可能相同？**

因為費用只看交易的資料大小，與金額無關。一筆簡單的 1 BTC 交易可能比複雜的 0.001 BTC 交易更便宜。
